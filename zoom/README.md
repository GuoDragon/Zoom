## 3) What does the agent return when a task does not pass?

Each task result line in `eval_details_*.jsonl` has these key fields:
- `verify_result`: final evaluator decision (authoritative for pass/fail)
- `result.success`: whether the agent thinks instruction execution completed
- `result.error`: execution error string when execution fails
- `result.final_message`: agent completion/explanation text
- `last_step_error`: last step error snapshot

Important: `result.success = true` can still pair with `verify_result = false`.
That means "agent self-reported done" but evaluator verification failed.
